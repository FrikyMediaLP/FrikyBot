Introduction

The FrikyBot is a combined Interface for you Twitch Streams. Chat Moderation, Collecting Stats and Handling advanced API Integration. Packages expand the Core Features of the Bot and provide even more ways to controll/influence your Streaming Experience.

The Bot currently only checks messages from ONE channel. Thats why you have to deploy this on your on Server/PC and need your own API Application and Twitch User Account.

Not all Core Modules need to be used, but some Packages require e.g. a Twitch Chat Connection. All Packages are optional, some are even only to be used by developers.

The Bot is currently still in Development, so some Features might be missing/unfinished. Please share your thoughts and help improve the Bot. I´d also like to say, that i dont actually know much of Server Handling/... I do this for a Hobby and keep learning along the way. Some parts like Storing UserData/... is still a thing i´d like to avoid! I just dont want to be responsible for leaks. But we will see how much I can grow this ;)

Thanks for reading this and even using the Bot. Means a lot and I try to make it as User friendly as possible.

Now please go ahead and setup your Bot :)

Introduction

The WebApp handles the Webpage/WebService you are using right NOW!

From Interfaces, APIs and Settings - everything is passed through the WebApp at some point. Thats why Authentication is a big part and we actually have multiple Versions of it (more about that later).

The Webserver is mounted to your localhost at a specific PORT. You can change this in the following settings, but remember: Everything is mounted to this Port (Settings, Cookies, ...)! So, if you change the Port, remember to change/update every other related Setting.

If your not using the Twith API Module, Authentication/Authorization is handled by the WebApp itself. This isn´t really recomended as Twitch is just way better at handling User Access/Storing them. Thats why we ADVICE YOU TO use the Twitch Authenticator later in the Settings.

Change Hostname (opt)

This Hostname refrences where Data from secured Twitch Services (anything over https) should be send to. Change this to your deployed Domain or your proxy adress, if you use any.

Leave this as "localhost" when you run your bot locally, but remember that Extern Services wont be able to send you secured data, since they dont know where/who you are!

Change Websocket Port (opt)

You can change the Port the Webserver is mounted on here.

Remember: Everything is mounted to this Port (Settings, Cookies, ...)! So, if you change the Port, remember to change/update every other related Setting.

Change Upload Limit (opt)

Changing the Upload Limit of Images/Sounds/Videos/etc. might decrease Performance.

Adjust TCP Ping-Pong Interval (opt)

Overlays and other Services that react on realtime events use the TCP Protocoll to exchange data. Most browsers/proxy servers close these connections automatically after a while, thats why the Server will frequently ping the Client to maintain a active connection.

Match this setting to your browsers or proxys setting (chrome and nginx use a timeout of 60s)! Otherwise these Clients might get disconnected and need to be reloaed!

Hostname

This is just a dummy Text, used by Packages an Modules to redirect Webhooks back to the deployed Server location. This doesnt change the Servers WebApp Deployment! Keep in mind to update the Twitch Application URL and use the 'Update EventSubs Button' in the Module Overview to update requested EventSubs to the new URL!

Codes and Secret

Authentication Codes are generated using Javascript Web Tokens (JWT). They are based on a Token-Secret, you can change/regenerate a new Secret, BUT you have to log in again, since the Token also changes.

Since all generated Codes have no expiration dates, changing the Secret is the only way to revoke a token.

Introduction

The Twitch IRC connects to the Chat and displays Messages, Events, ... All Messages (Events,...) can be reveived without a supplied User, BUT to send Messages (e.g. for Commands) you need a BotUser Account (can also be the Broadcaster or an other). This Account can be connecting by loggin in using the FrikyBot Twitch API Module or by an OAuth Token generated by an external Service.

The FrikyBot also directly comes with the FrankerFaceZ and BTTV APIs. Theses are nativly supported by all Core Modules and provide even more Capabilities in Chat Moderation and Data Collection.

Login using Twitch

Login using Twitch requieres Setup of the Twitch API! If you havent already, pls setup your Twitch Dev Application first!

User Login

To be able to send Messages/Moderate Chat/... the FrikyBot requieres a Twitch User Account. This can be your own or a second one created as the Bot User.

If you have Twitch API already set up, you can directly Log in using Twitch with the Account of your Choice. Otherwise you can use a third Party Service to obtain a Twitch OAuth Token and use that one.

Token Origin / Status

The Token needed to authenticate you to the Twitch Chat isn´t requiered to originate from your Twitch API Application (thats why you can use third Party Sevices), BUT if you change any Setting to the Twitch API Application or your third Party Service might force you to regenerate this OAuth Token!

Login using Twitch

Make sure your Twitch Dec Application is setup and valid. A Token with rights to "read/write Chat Messages", "moderate Chat" and "read/write Whispers" will be requested, NOTHING MORE!

BUT, unless you give us permission to check your Subscriptions too, we wont be able to check the Bots (Sub)Emotes and cant track them! Twitch currently doesnt support looking up an individuals Subscription - we hope they add this soon!

This Login will redirect you to Twitch and then back here (you MIGHT need to navigate to this step again tho)

Introduction

The Twitch API is used to get extra information about Users/Streams/Games/... This requires you to set up your own Twitch Dev Application and Bot Client. How to do this is show on the next pages.

API Access is split into two Tiers: AppAccess and UserAccess. User Access expandes the Bots authorization to acces your User specific privileges (Setting Titles/Games, access User Settings/...).

IMPORTANT

The FrikyBot WebInterface also uses your Twitch Accounts to Authenticate you on this site. YOU HAVE TO ADD an Admin User to access settings and other interfaces!

Twitch Dev Application

To Access the Twitch API you need a to register your own Twitch Dev Application and fill in the following Information.


Client ID

A Public ID of your Twitch Dev Application.


Client Secret

A Private Token used to request Access Tokens for you Application. YOU SHOULDNT SHARE THIS, but ... we need it


Redirect URL

You have to set your Redirect URL in your Application Settings to redirect Logins using Twitch to the Bot.

Remember to update this URL every time you change the Hostname (or Port when using localhost). NOTE: The Redirect URL is CASE SENSITIVE! Make sure you match it EXACTLY!

EventSub Topics

EventSubs are the way Twitch can relay events using Webhooks to an Application/Bot. Using these enables a realtime tracking of Subs, Stream Updates and Chat Events.

All of these are used by default and are very usefull - but you can turn them off if you want to.

For more information visit the Twitch EventSub Documentation!

Introduction

Packages expand the Core FrikyBot Modules into powerfull and creative Features. This Packages can vary from simple Commands to all out Chat Games and User Interfaces.

Packages are optional, but Packages provide the actual Features of the Bot. If you want to create your own Package, check the Docs for more information.

Adding Packages

The Bot auto-detects Packages placed in the Packages Folder, but you can also add a Package by simply entering the Packages Name. Since there is currently no public archive for ALL Packages, you can enter anything here. Only on Boot/Restart will these Package Names be checked for what even is currently installed.

Packages provide their own Settings/Configs. Here you are only installing a Package to the Bots Boot Option.
