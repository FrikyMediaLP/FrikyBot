Introduction

The FrikyBot is a combined Interface for you Twitch Streams. Chat Moderation, Collecting Stats and Handling advanced API Integration. Packages expand the Core Features of the Bot and provide even more ways to controll/influence your Streaming Experience.

The Bot currently only checks messages from ONE channel. Thats why you have to deploy this on your on Server/PC and need your own API Application and Twitch User Account.

Not all Core Modules need to be used, but some Packages require e.g. a Twitch Chat Connection. All Packages are optional, some are even only to be used by developers.

The Bot is currently still in Development, so some Features might be missing/unfinished. Please share your thoughts and help improve the Bot. I´d also like to say, that i dont actually know much of Server Handling/... I do this for a Hobby and keep learning along the way. Some parts like Storing UserData/... is still a thing i´d like to avoid! I just dont want to be responsible for leaks. But we will see how much I can grow this ;)

Thanks for reading this and even using the Bot. Means a lot and I try to make it as User friendly as possible.

Now please go ahead and setup your Bot :)

Introduction

The WebApp handles the Webpage/WebService you are using right NOW!

From Interfaces, APIs and Settings - everything is passed through the WebApp at some point. Thats why Authentication is a big part and we actually have multiple Versions of it (more about that later).

The Webserver is mounted to your localhost at a specific PORT. You can change this in the following settings, but remember: Everything is mounted to this Port (Settings, Cookies, ...)! So, if you change the Port, remember to change/update every other related Setting.

If your not using the Twith API Module, Authentication/Authorization is handled by the WebApp itself. This isn´t really recomended as Twitch is just way better at handling User Access/Storing them. Thats why we ADVICE YOU TO use the Twitch Authenticator later in the Settings.

FrikyBot Authentication

Authentication/Authorization is needed to access sensitive Data or change settings. I advice you to use the Twitch API Authenticator instead of my own tho!! (more about that later)

Codes and Secret

Authentication Codes are generated using Javascript Web Tokens (JWT). They are based on a Token-Secret, you can change/regenerate a new Secret, BUT you have to log in again, since the Token also changes.

Since all generated Codes have no expiration dates, changing the Secret is the only way to revoke a token.

Introduction

The Twitch API is used to get extra information about Users/Streams/Games/... This requires you to set up your own Twitch Dev Application and Bot Client. How to do this is show on the next pages.

API Access is split into two Tiers: AppAccess and UserAccess. User Access expandes the Bots authorization to acces your User specific privileges (Setting Titles/Games, access User Settings/...).

IMPORTANT

The FrikyBot WebInterface also uses your Twitch Accounts to Authenticate you on this site. YOU HAVE TO ADD an Admin User to access settings and other interfaces!

Twitch Dev Application

To Access the Twitch API you need a to register your own Twitch Dev Application and fill in the following Information.


Client ID

A Public ID of your Twitch Dev Application.


Client Secret

A Private Token used to request Access Tokens for you Application. YOU SHOULDNT SHARE THIS, but ... we need it


Redirect URL

You have to set your Redirect URL in your Application Settings to redirect Logins using Twitch to the Bot.

Remember to update this URL every time you change the FrikyBot Webserver Port!!

Introduction

DataCollection is a big part of the FrikyBot. This mainly refers to Chat Message Data(Active Chatters, Fav. Emotes, ...) and Stream Data(Viewer Counts, Sub History, Most played Games, ...).

All Data the Bot "collects" is open to anyone in the Chat or authorized for the Twitch API. So no private Data is collected.

DataCollection is currently still WIP and not yet implemented. More about it in the future.

Introduction

Packages expand the Core FrikyBot Modules into powerfull and creative Features. This Packages can vary from simple Commands to all out Chat Games and User Interfaces.

Packages are optional, but Packages provide the actual Features of the Bot. If you want to create your own Package, check the Docs for more information.

Adding Packages

The Bot auto-detects Packages placed in the Packages Folder, but you can also add a Package by simply entering the Packages Name. Since there is currently no public archive for ALL Packages, you can enter anything here. Only on Boot/Restart will these Package Names be checked for what even is currently installed.

Packages provide their own Settings/Configs. Here you are only installing a Package to the Bots Boot Option.
